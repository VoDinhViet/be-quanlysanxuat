# Suppliers Module Spec

## Purpose

The suppliers module owns supplier master data used by procurement, outside processing, and purchase flows.

Source:

```text
src/api/suppliers/
```

## Public API

All suppliers endpoints are authenticated and require permissions.

### GET /suppliers

Decorator:

- `@ApiAuth(..., isPaginated: true)`
- `@Permissions('suppliers:read')`

Query DTO:

- `GetSuppliersReqDto`

Query fields inherited from `PageOptionsDto`:

- `page`
- `limit`
- `q`
- `order`

Response:

- `OffsetPaginatedDto<SupplierResDto>`

Response fields:

- `id`
- `code`
- `name`
- `email`
- `phoneNumber`
- `address`
- `createdAt`
- `updatedAt`

Business rules:

- Supports pagination.
- Supports keyword search by `code`, `name`, `email`, `phoneNumber`, and `address`.
- Sorts by `createdAt` using `PageOptionsDto.order`.
- Excludes soft-deleted rows with `deletedAt IS NULL`.

### POST /suppliers

Decorator:

- `@ApiAuth(..., statusCode: HttpStatus.CREATED)`
- `@Permissions('suppliers:create')`

Request DTO:

- `CreateSupplierReqDto`

Request fields:

- `name`
- `email`
- `phoneNumber`
- `address`

Response:

- `SupplierResDto`

Business rules:

- `name` is required.
- `email`, `phoneNumber`, and `address` are optional.
- Supplier code is generated by service with `NCC` prefix.

### PATCH /suppliers/:supplierId

Decorator:

- `@ApiAuth(...)`
- `@Permissions('suppliers:update')`

Params:

- `supplierId` through `@UUIDParam('supplierId')`

Request DTO:

- `UpdateSupplierReqDto`

Business rules:

- Supplier must exist and not be soft-deleted.
- Set `updatedAt` explicitly.
- Return the updated supplier.

Errors:

- Missing supplier: `ErrorCode.E002`, HTTP `404`.

### DELETE /suppliers/:supplierId

Decorator:

- `@ApiAuth(...)`
- `@Permissions('suppliers:delete')`

Params:

- `supplierId` through `@UUIDParam('supplierId')`

Business rules:

- Supplier must exist and not be soft-deleted.
- Soft delete by setting `deletedAt` and `updatedAt`.
- Return the deleted supplier.

Errors:

- Missing supplier: `ErrorCode.E002`, HTTP `404`.

## Permissions

Current supplier permissions:

- `suppliers:read`
- `suppliers:create`
- `suppliers:update`
- `suppliers:delete`
- `suppliers:manage`

Current endpoint usage:

- `GET /suppliers`: `suppliers:read`
- `POST /suppliers`: `suppliers:create`
- `PATCH /suppliers/:supplierId`: `suppliers:update`
- `DELETE /suppliers/:supplierId`: `suppliers:delete`

## Dependencies

- Drizzle database client through `DRIZZLE`
- `suppliers` schema
- `PageOptionsDto`
- `OffsetPaginatedDto`
- `OffsetPaginationDto`
- `plainToInstance` response mapping

## Database Rules

- Use `suppliers` schema from `src/database/schemas/suppliers.ts`.
- Default reads must exclude soft-deleted rows.
- Use Drizzle operators from `drizzle-orm`.
- Use `.returning()` when insert/update/delete needs the changed row.
- Set `updatedAt` explicitly on updates and soft deletes.
- Use soft delete through `deletedAt`; do not hard delete suppliers without explicit approval.

## Security Rules

- Protected endpoints must keep permission decorators.
- Validate route params with `@UUIDParam('supplierId')` when detail/write endpoints are added.
- Validate request body fields through DTO decorators when create/update endpoints are added.
- Do not accept raw sort fields from clients without a whitelist.
